The Io Ai REST API gives you programmatic access to your documents, projects, and publishing workflows. Every request is made over HTTPS to a versioned base URL, and every request and response body uses JSON. You can use the API to create and update documentation content, trigger publishing, and receive real-time events via webhooks.

## Base URL

All API endpoints are relative to the following base URL:

```
https://api.io-oi.ai/v1
```

## Request and response format

All requests must include the `Content-Type: application/json` header when sending a body. All responses return JSON, including error responses.

## Authentication

The API uses Bearer token authentication. You pass your API key in the `Authorization` header of every request. See the [Authentication](/api-reference/authentication) page for instructions on generating and managing API keys.

## Versioning

The current API version is **v1**, reflected in the base URL path. When breaking changes are introduced, a new version path (e.g., `/v2`) will be released. The previous version will continue to function for a deprecation period announced in advance. Non-breaking additions — such as new optional fields or new endpoints — may be made to the current version without a version bump.

## Rate limiting

Requests are rate-limited per API key. If you exceed the limit, you will receive a `429 Too Many Requests` response. See the [rate limits section](/api-reference/authentication#rate-limits) on the Authentication page for plan-specific limits and retry guidance.

## Error responses

When a request fails, the API returns an appropriate HTTP status code along with a JSON body containing an `error` code and a human-readable `message`.

```json
{
  "error": "not_found",
  "message": "The requested document could not be found."
}
```

Common HTTP status codes you will encounter:

| Status | Meaning |
|--------|---------|
| `400` | Bad request — invalid or missing parameters |
| `401` | Unauthorized — missing or invalid API key |
| `403` | Forbidden — your key does not have the required scope |
| `404` | Not found — the resource does not exist |
| `429` | Too many requests — rate limit exceeded |
| `500` | Internal server error — something went wrong on our end |
