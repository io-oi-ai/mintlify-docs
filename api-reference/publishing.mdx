---
title: "Publishing API: publish, unpublish, and check status"
description: "Use the Io Ai publishing API to publish or unpublish documents and projects, override custom domains, and poll live publish status."
---

The publishing endpoints let you control the public visibility of your documentation. You can publish a single document or an entire project at once, specify a custom domain, check whether content is live, and unpublish when needed.

<Note>
After a successful publish request, it may take up to 30 seconds for changes to propagate across the Io Ai CDN. If you need to confirm that content is live before sending users to it, poll the [Check publish status](#check-publish-status) endpoint until it returns `"status": "published"`.
</Note>

---

## Publish a document or project

Publishes a document or an entire project, making it publicly accessible. You must supply either `document_id` or `project_id` — not both.

**POST** `/publish`

### Body parameters

<ParamField body="document_id" type="string">
  The ID of the document to publish. Required if `project_id` is not provided.
</ParamField>

<ParamField body="project_id" type="string">
  The ID of the project to publish. When you publish a project, all documents with `status: published` within it become publicly accessible. Required if `document_id` is not provided.
</ParamField>

<ParamField body="custom_domain" type="string">
  An optional domain to override the project's default custom domain for this publish operation (for example, `"docs.example.com"`). If omitted, the project's configured custom domain is used.
</ParamField>

### Response fields

<ResponseField name="url" type="string" required>
  The public URL where the published content is accessible.
</ResponseField>

<ResponseField name="published_at" type="string" required>
  ISO 8601 timestamp of when the content was published.
</ResponseField>

### Example

<CodeGroup>

```bash cURL
curl -X POST "https://api.io-oi.ai/v1/publish" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "document_id": "doc_xyz789"
  }'
```

```javascript JavaScript
const response = await fetch("https://api.io-oi.ai/v1/publish", {
  method: "POST",
  headers: {
    Authorization: "Bearer YOUR_API_KEY",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    document_id: "doc_xyz789",
  }),
});
const result = await response.json();
// { url: "https://...", published_at: "2026-05-10T12:00:00Z" }
```

</CodeGroup>

## Unpublish a document

Removes a document from public access. The document's content is preserved in your account but is no longer accessible via its public URL.

**DELETE** `/publish/:document_id`

### Path parameters

<ParamField path="document_id" type="string" required>
  The ID of the document to unpublish.
</ParamField>

### Response

Returns `204 No Content` on success with an empty response body.

### Example

<CodeGroup>

```bash cURL
curl -X DELETE "https://api.io-oi.ai/v1/publish/doc_xyz789" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
await fetch("https://api.io-oi.ai/v1/publish/doc_xyz789", {
  method: "DELETE",
  headers: {
    Authorization: "Bearer YOUR_API_KEY",
  },
});
```

</CodeGroup>

## Check publish status

Returns the current publish status of a document, including its public URL and the time it was last published.

**GET** `/publish/status/:document_id`

### Path parameters

<ParamField path="document_id" type="string" required>
  The ID of the document whose publish status you want to check.
</ParamField>

### Response fields

<ResponseField name="status" type="string" required>
  Current publish state of the document: `published`, `draft`, or `pending`. A `pending` status means a publish operation is in progress.
</ResponseField>

<ResponseField name="url" type="string">
  The public URL of the document. Present only when `status` is `published`.
</ResponseField>

<ResponseField name="published_at" type="string">
  ISO 8601 timestamp of when the document was last published. Present only when `status` is `published`.
</ResponseField>

### Example

<CodeGroup>

```bash cURL
curl "https://api.io-oi.ai/v1/publish/status/doc_xyz789" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch(
  "https://api.io-oi.ai/v1/publish/status/doc_xyz789",
  {
    headers: {
      Authorization: "Bearer YOUR_API_KEY",
    },
  }
);
const status = await response.json();
// { status: "published", url: "https://...", published_at: "2026-05-10T12:00:00Z" }
```

</CodeGroup>
