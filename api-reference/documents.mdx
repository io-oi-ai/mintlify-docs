---
title: "Documents API: create, update, and delete content"
description: "Create, list, retrieve, update, and delete documents with the Io Ai REST API. Full reference with parameters, response fields, and code examples."
---

Documents are the core content objects in Io Ai. Each document belongs to a project and contains a Markdown body that Io Ai renders into a plain-language documentation page. The endpoints below let you manage the full lifecycle of a document — from creation through publication and deletion.

---

## List documents

Retrieves all documents within a project, with optional filtering by status and pagination controls.

**GET** `/documents`

### Query parameters

<ParamField query="project_id" type="string" required>
  The ID of the project whose documents you want to list.
</ParamField>

<ParamField query="limit" type="number" default="20">
  Maximum number of documents to return per page. Accepts values between 1 and 100.
</ParamField>

<ParamField query="offset" type="number" default="0">
  Number of documents to skip before returning results. Use with `limit` for pagination.
</ParamField>

<ParamField query="status" type="string">
  Filter by document status. Accepts `draft` or `published`.
</ParamField>

### Response fields

<ResponseField name="data" type="object[]">
  Array of document summary objects.
  <Expandable title="properties">
    <ResponseField name="id" type="string" required>
      Unique identifier for the document.
    </ResponseField>
    <ResponseField name="title" type="string" required>
      Human-readable title of the document.
    </ResponseField>
    <ResponseField name="status" type="string" required>
      Current status: `draft` or `published`.
    </ResponseField>
    <ResponseField name="created_at" type="string" required>
      ISO 8601 timestamp of when the document was created.
    </ResponseField>
    <ResponseField name="updated_at" type="string" required>
      ISO 8601 timestamp of the last update.
    </ResponseField>
  </Expandable>
</ResponseField>

### Examples

<CodeGroup>

```bash cURL
curl "https://api.io-oi.ai/v1/documents?project_id=proj_abc123&limit=10&status=published" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch(
  "https://api.io-oi.ai/v1/documents?project_id=proj_abc123&limit=10&status=published",
  {
    headers: {
      Authorization: "Bearer YOUR_API_KEY",
    },
  }
);
const data = await response.json();
```

</CodeGroup>

## Get a document

Retrieves a single document by its ID, including the full Markdown content body.

**GET** `/documents/:id`

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the document to retrieve.
</ParamField>

### Response fields

<ResponseField name="id" type="string" required>
  Unique identifier for the document.
</ResponseField>

<ResponseField name="title" type="string" required>
  Human-readable title of the document.
</ResponseField>

<ResponseField name="content" type="string" required>
  Full document body as a Markdown string.
</ResponseField>

<ResponseField name="status" type="string" required>
  Current status: `draft` or `published`.
</ResponseField>

<ResponseField name="project_id" type="string" required>
  ID of the project this document belongs to.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the document was created.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 timestamp of the last update.
</ResponseField>

### Examples

<CodeGroup>

```bash cURL
curl "https://api.io-oi.ai/v1/documents/doc_xyz789" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch(
  "https://api.io-oi.ai/v1/documents/doc_xyz789",
  {
    headers: {
      Authorization: "Bearer YOUR_API_KEY",
    },
  }
);
const document = await response.json();
```

</CodeGroup>

## Create a document

Creates a new document in the specified project.

**POST** `/documents`

### Body parameters

<ParamField body="project_id" type="string" required>
  The ID of the project to create the document in.
</ParamField>

<ParamField body="title" type="string" required>
  The title of the new document.
</ParamField>

<ParamField body="content" type="string">
  The document body as a Markdown string. Can be empty on creation.
</ParamField>

<ParamField body="status" type="string" default="draft">
  Initial status of the document. Accepts `draft` or `published`.
</ParamField>

### Response fields

Returns the created document object with the same fields as [Get a document](#get-a-document).

### Examples

<CodeGroup>

```bash cURL
curl -X POST "https://api.io-oi.ai/v1/documents" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "proj_abc123",
    "title": "Getting started",
    "content": "# Getting started\n\nWelcome to the guide.",
    "status": "draft"
  }'
```

```javascript JavaScript
const response = await fetch("https://api.io-oi.ai/v1/documents", {
  method: "POST",
  headers: {
    Authorization: "Bearer YOUR_API_KEY",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    project_id: "proj_abc123",
    title: "Getting started",
    content: "# Getting started\n\nWelcome to the guide.",
    status: "draft",
  }),
});
const created = await response.json();
```

</CodeGroup>

## Update a document

Updates one or more fields of an existing document. All body parameters are optional — include only the fields you want to change.

**PUT** `/documents/:id`

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the document to update.
</ParamField>

### Body parameters

<ParamField body="title" type="string">
  New title for the document.
</ParamField>

<ParamField body="content" type="string">
  Updated Markdown body for the document.
</ParamField>

<ParamField body="status" type="string">
  New status for the document. Accepts `draft` or `published`.
</ParamField>

### Response fields

Returns the updated document object with the same fields as [Get a document](#get-a-document).

### Examples

<CodeGroup>

```bash cURL
curl -X PUT "https://api.io-oi.ai/v1/documents/doc_xyz789" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Getting started (revised)",
    "status": "published"
  }'
```

```javascript JavaScript
const response = await fetch(
  "https://api.io-oi.ai/v1/documents/doc_xyz789",
  {
    method: "PUT",
    headers: {
      Authorization: "Bearer YOUR_API_KEY",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      title: "Getting started (revised)",
      status: "published",
    }),
  }
);
const updated = await response.json();
```

</CodeGroup>

## Delete a document

Permanently deletes a document. This action cannot be undone.

**DELETE** `/documents/:id`

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the document to delete.
</ParamField>

### Response

Returns `204 No Content` on success with an empty response body.

### Examples

<CodeGroup>

```bash cURL
curl -X DELETE "https://api.io-oi.ai/v1/documents/doc_xyz789" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
await fetch("https://api.io-oi.ai/v1/documents/doc_xyz789", {
  method: "DELETE",
  headers: {
    Authorization: "Bearer YOUR_API_KEY",
  },
});
```

</CodeGroup>
