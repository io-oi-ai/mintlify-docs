---
title: "Configure project and account settings in Io Ai"
description: "A complete reference for every setting in Io Ai — from your account profile and notification preferences to per-project editor and visibility options."
---

Every aspect of how Io Ai behaves — from your personal display name to the default language of a new document — is controlled through settings. You can reach account-level settings from your avatar menu in the top-right corner, and project-level settings from the **Settings** tab inside any project. The sections below describe each setting and what it controls.

<Note>
  Settings marked with an **Owner** badge can only be changed by a project Owner. If you do not see a setting described here, ask your project Owner to update it for you.
</Note>

<AccordionGroup>
  <Accordion title="Account settings" icon="user">
    Account settings apply to your personal Io Ai profile across all projects you belong to.

    | Setting | Description |
    |---------|-------------|
    | **Display name** | The name shown on your comments, activity feed, and collaborator list. |
    | **Email address** | The address used for login and all notification emails. Changing your email triggers a verification message to the new address. |
    | **Password** | Update your password at any time. You must enter your current password to confirm the change. |
    | **Notification preferences** | Choose which events trigger email or in-app notifications. See the [Notification settings](#notification-settings) section for the full list. |

    To update any account setting, open your avatar menu and select **Account settings**.
  </Accordion>

  <Accordion title="Project settings" icon="folder" defaultOpen={false}>
    Project settings control how a specific project behaves and who can see it. Only project Owners can change these settings.

    | Setting | Description |
    |---------|-------------|
    | **Project name** | The title displayed in your dashboard and in the browser tab when visitors view your docs. |
    | **Description** | A short summary of the project shown on your dashboard and in search results. |
    | **Visibility** | Set the project to **Public** (anyone with the link can read it) or **Private** (only invited collaborators can access it). |
    | **Default language** | The locale used for spell check, hyphenation, and date formatting across all documents in the project. |
    | **Analytics** | Enable or disable the built-in page-view analytics for your published docs site. When enabled, you can view traffic data from the **Analytics** tab. |

    <Note>
      Changing a project from Public to Private immediately restricts access. Anyone who had the link will lose access until they are added as a collaborator.
    </Note>
  </Accordion>

  <Accordion title="Editor settings" icon="pen-line" defaultOpen={false}>
    Editor settings control the default behavior of the Io Ai document editor. You can set defaults at the project level, and individual collaborators can override them for their own session.

    | Setting | Description |
    |---------|-------------|
    | **Default document format** | Choose between **Markdown** (`.md`) and **MDX** (`.mdx`) for new documents created in this project. MDX enables Mintlify components and JSX. |
    | **Autosave interval** | How frequently the editor saves your work automatically. Options are every **30 seconds**, **1 minute**, **2 minutes**, or **Off** (manual save only). |
    | **Spell check** | Enable or disable browser-native spell checking in the editor. This uses your operating system's dictionary and respects the project's default language setting. |

    <Tip>
      If you write technical content with many code terms that trigger false spell-check warnings, disable spell check at the project level and use an external linter instead.
    </Tip>
  </Accordion>

  <Accordion title="Notification settings" icon="bell" defaultOpen={false}>
    Notification settings let you choose which events in Io Ai send you an email. You can configure these globally in **Account settings** or override them per project.

    | Event | Description |
    |-------|-------------|
    | **New comment** | Receive an email when a collaborator leaves a comment on any document you have access to. |
    | **Comment mention** | Receive an email only when someone mentions you directly with `@yourname` in a comment. |
    | **Document published** | Receive an email whenever a document in this project is published or republished. |
    | **Collaborator joined** | Receive an email when a new collaborator accepts an invitation to the project. |
    | **Collaborator role changed** | Receive an email when an Owner changes your role or another collaborator's role. |

    To mute all notifications for a single project without changing your global preferences, open **Project settings** and toggle **Email notifications** off.
  </Accordion>
</AccordionGroup>
