Every document in Io Ai lives inside a project. Before you create a document, make sure you have a project set up — if you haven't done that yet, see the [quick start guide](/quickstart). Once your project is ready, creating and organizing documents takes just a few minutes.

## Create a new document

<Steps>
  <Step title="Open your project dashboard">
    Sign in to Io Ai and select the project where you want to add a document. The dashboard lists all existing documents in that project.
  </Step>
  <Step title="Click New document">
    Select **New document** in the top-right corner of the dashboard. Io Ai opens a blank document in the editor and prompts you to give it a title.
  </Step>
  <Step title="Add a title">
    Type a clear, descriptive title in the title field at the top of the editor. The title becomes the page heading and the document's name in the dashboard. Keep it short and specific — for example, "Set up two-factor authentication" rather than "Security".
  </Step>
  <Step title="Write your content">
    Start typing in the editor body. Io Ai supports Markdown and MDX, so you can use familiar syntax for headings, lists, bold and italic text, links, and code blocks. See [formatting basics](#formatting-basics) below for a quick reference.
  </Step>
  <Step title="Save or publish">
    Use **Save draft** to preserve your progress without making the document visible to readers. When the content is ready, click **Publish** to make it live. For more detail on publishing, see [Publish your documentation](/guides/publishing).
  </Step>
</Steps>

## The document editor

The editor is a full-screen writing environment with two modes you can switch between:

- **Write mode** — a clean text surface with Markdown syntax highlighting and a formatting toolbar.
- **Preview mode** — a rendered view that shows exactly how your document will look when published.

The toolbar at the top of the editor gives you quick access to formatting actions. You can also type Markdown directly — both approaches produce the same output.

<Tip>
Use **Preview mode** before publishing to catch formatting issues and check that all links and code blocks render correctly.
</Tip>

## Formatting basics

Io Ai uses standard Markdown syntax. Here are the most common elements you'll use:

**Headings** — Use `#` for a top-level heading, `##` for a second-level heading, and `###` for a third level. Keep headings short and descriptive so readers can scan your document quickly.

**Bold and italic** — Wrap text in `**double asterisks**` for bold and `*single asterisks*` for italic. Use bold for UI element names and key terms. Use italic sparingly.

**Bulleted and numbered lists** — Start lines with `-` for bullets or `1.` for numbered steps. Use numbered lists for sequential instructions where order matters.

**Links** — Use `[link text](url)` for external links or `[link text](/path/to-page)` for internal ones. Always use root-relative paths for internal links.

**Code blocks** — Wrap inline code in single backticks: `` `code` ``. For multi-line code, use triple backticks and specify the language:

````markdown
```javascript
const greeting = 'Hello, world!';
console.log(greeting);
```
````

## Organize documents into folders

As your project grows, use folders to group related documents.

<Steps>
  <Step title="Create a folder">
    In the project dashboard, click **New folder** and give it a descriptive name. Use noun phrases that reflect the content — for example, "Getting started", "API reference", or "Troubleshooting".
  </Step>
  <Step title="Move documents into the folder">
    Drag and drop documents from the dashboard into the folder, or open a document, click **Move**, and select the destination folder.
  </Step>
  <Step title="Reorder documents">
    Within a folder, drag documents into the order you want them to appear in your published navigation. Io Ai reflects that order in the sidebar automatically.
  </Step>
</Steps>

<Note>
Folders map directly to sections in your published documentation's sidebar. A well-organized folder structure helps readers find content faster, so take a moment to plan your hierarchy before creating many documents.
</Note>

## Use document templates

Templates give you a pre-structured starting point so you don't have to build common document types from scratch.

<Steps>
  <Step title="Open the template picker">
    When creating a new document, click **Start from template** instead of opening a blank editor.
  </Step>
  <Step title="Choose a template">
    Browse the available templates — for example, "How-to guide", "API endpoint", "Release notes", or "Troubleshooting article". Select the one that best matches the type of content you're writing.
  </Step>
  <Step title="Customize the template">
    Replace the placeholder text with your content. The template structure guides you through the sections a reader expects in that document type.
  </Step>
</Steps>

## Saving drafts vs. publishing

<CardGroup cols={2}>
  <Card title="Draft" icon="floppy-disk">
    Saves your work and keeps the document private. Only project members can see drafts. Use drafts while writing, editing, or waiting for review.
  </Card>
  <Card title="Published" icon="globe">
    Makes the document visible to anyone with access to your published documentation site. You can update a published document at any time and republish to push changes live.
  </Card>
</CardGroup>

Io Ai autosaves your content every few seconds, so you won't lose work if you close the browser. The autosave indicator in the top bar shows the last saved time.

## Tips for structuring content clearly

- **Start with the goal.** Open each document with one sentence that tells the reader what they will accomplish or understand by the end.
- **Use headings to signal sections.** Readers scan before they read. Clear headings help them jump to the part they need.
- **One idea per paragraph.** Short, focused paragraphs are easier to read than long blocks of text.
- **Front-load important information.** Put the most critical information first within each section — don't bury key steps or warnings at the end.
- **Use the AI assistant.** If you're not sure how to phrase something, use Io Ai's AI writing assistant to simplify or restructure your text. See [Write better docs with Io Ai's AI assistant](/guides/ai-writing).
